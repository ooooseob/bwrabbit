## 1. 핵심 UX/UI 요소별 분석 및 개선 방안

데이터의 흐름과 사용자의 목표 달성에 초점을 맞춰 작성했다.

UX/UI의 8가지 핵심 요소를 기준으로 현재 프로젝트의 구조를 분석하고 개선 방향 도출.

### 1. 유용성 (Useful) & 가치성 (Valuable) 확보
목표: 사용자가 서비스를 사용하는 핵심 목표(예: 게임 성과 분석)를 가장 쉽고 빠르게 달성하도록 지원해야 한다.

*   분석: 데이터 집계와 분석 도구를 포함하고 잇어 가치성을 높이려면, 단순히 데이터를 보여주는 것을 넘어 '행동 유도'와 '통찰력'을 제공해야 한다.
*   개선 방향:
    *   핵심 가치 강조: 사용자가 보기에 가장 중요한 성과(예 : 매출상승?)를 메인화면에 보여주어 사용자가 이 서비스를 활요하면 목표 달성에 어떻게 기여하는지 명확히 설명해야 한다.
    *   맥락 제공: 단순한 수치 나열 대신, 수치 변화의 이유를 설명하는 텍스트(예: "지난주 대비 15% 상승, 긍정 키워드 증가")를 함께 제공하여 데이터의 유용성을 극대화 시킨다.

### 2. 사용성 (Usable) 확보
목표: 사용자가 복잡한 분석 기능이나 데이터 입력 과정에서 혼란을 느끼지 않고 목표를 달성하도록 직관적인 워크플로우를 구축해야 한다.

*   분석: 현재 이루어진 워크플로우는 첫 방문 사용자에게 친절하지 않은 느낌이다. 온보딩을 통해 사용자 경험을 높여야 한다.
*   개선 방향:
    *   단순화된 워크플로우: 단계별(Step-by-Step) 프로세스로 나누어 사용자가 한 번에 많은 정보를 처리하지 않도록 하거나, 예시 질문을 제공한다.
    *   명확한 피드백: 데이터 로딩 중이거나 분석 중일 때는 반드시 시각적인 로딩 인디케이터(스켈레톤 UI 등)와 명확한 텍스트 피드백을 제공하여 사용자의 대기 시간을 관리해야 한다.

### 3. 탐색 가능성 (Findable) 확보
목표: 사용자가 원하는 정보나 기능에 최소한의 클릭으로 접근할 수 있도록 정보 구조를 체계화해야 한다.

*   분석: 메뉴 구조(Navbar, Pagination)와 데이터 접근 방식이 중요.
*   개선 방향:
    *   강력한 검색 기능: 데이터셋이 확장될 경우, 필터링 및 검색 기능을 고도화하여 사용자가 원하는 특정 데이터 포인트에 즉시 도달할 수 있도록 해야 한다.

### 4. 신뢰성 (Credible) 확보
목표: 제공되는 모든 데이터와 분석 결과에 대한 사용자의 믿음을 확보해야 한다.

*   분석: 데이터의 출처(Source)와 분석 방법론(Methodology)이 신뢰의 핵심.
*   개선 방향:
    *   데이터 출처 명시: 표시되는 모든 지표 옆에 데이터가 어디서 왔는지(예: Steam 리뷰, 유튜브 시그널 등)를 명시하여 데이터의 신뢰성을 높여야 한다.
    *   투명성: 분석 알고리즘이나 모델의 작동 방식에 대한 간략한 설명을 제공하여, 결과가 어떻게 도출되었는지에 대한 투명성을 확보한다.

### 5. 사용성 테스트 & 지속적 개선 (UCD 접근법)
목표: 사용자의 실제 행동과 피드백을 통해 지속적으로 제품을 개선해야 한다.

*   분석: 현재는 정적인 구조만 존재해서 개선은 반복적인 테스트를 통해 이루어져야 한다.
*   개선 방향:
    *   A/B 테스트 준비: 특정 레이아웃, 버튼 위치, 데이터 표시 방식에 대해 A/B 테스트를 설계할 수 있도록 컴포넌트화(Componentization)를 철저히 준비해야한다.
    *   피드백 루프 구축: 사용자가 특정 분석 결과를 보고 "이 데이터가 부족하다"고 느낄 때, 이를 쉽게 피드백할 수 있는 기능을 추가하는 것도 생각헤야 한다.

### 6. UI 일관성 & 디자인 시스템 구축
목표: 모든 화면에서 동일한 경험을 제공하여 인지 부하를 줄이고 브랜드의 통일성을 확보해야 한다.

*   분석: 현재 컴포넌트가 존재하므로, 이들을 기반으로 디자인 시스템을 구축하는 것이 중요하다. (현재 디자인 시스템이 없으므로)
*   개선 방향:
    *   디자인 시스템 구축: 색상 팔레트, 타이포그래피 스케일, 간격, 버튼 스타일 등에 대한 명확한 디자인 토큰을 정의하고 이를 Theme 또는 Style 파일에 통합해야 한다. (현재 스타일 파일 수정)
    *   컴포넌트 표준화: GameCard와 같은 핵심 컴포넌트의 디자인을 표준화하여, 모든 페이지에서 동일한 시각적 언어를 사용하도록 강제해야 한다.

### 7. 접근성 (Accessible) 확보
목표: 모든 사용자가 물리적 제약 없이 콘텐츠에 접근하고 상호작용할 수 있도록 설계해야 한다.

*   분석: 이는 시각적 요소뿐만 아니라 코드 레벨의 고려가 필요하다.
*   개선 방향:
    *   색상 대비(Contrast): CSS에서 텍스트와 배경 간의 대비 비율(WCAG 기준)을 반드시 확인해야 한다. (WebAIM에서 5점 이상 받아야 보통이라고 볼 수 있는 걸로 안다.)
    *   키보드 내비게이션: 모든 상호작용 요소가 키보드(Tab, Enter)로 접근 가능하도록 명확히 해야하고 스크린 리더가 읽어주는 상황도 고려해야 한다.(예 : aria 속성 사용)

### 8. 감성 UX (Emotional Design) 확보
목표: 사용자와 시스템 간의 상호작용에서 긍정적인 감정(만족감, 몰입감)을 유발해야 한다.

*   분석: 데이터 분석 앱은 특히 몰입감과 성취감을 제공할 때 감성적 연결이 강해진다.
*   개선 방향:
    *   미묘한 애니메이션: 데이터가 로드되거나 업데이트될 때, 갑작스러운 변화 대신 부드럽고 의미 있는 애니메이션(예: 추후 생길 차트 데이터가 점진적으로 채워지는 효과)을 사용하여 몰입도를 높여야 한다.
    *   긍정적 피드백: 목표를 달성했을 때 명확한 시각적 피드백을 제공하여 성취감을 강화해야 한다.

## 2. 간단한 요약 및 진행해야 될 사항(주관적)

결국 사용자가 서비스를 처음 접했을 때, 이 도구가 나에게 어떤 가치를 제공하고, 어떻게 사용해야 가장 큰 성취감을 얻을 수 있는지를 가장 빠륵 매력적으로 전달하는 것이 목표다.

### 1. 사용자의 시선을 사로잡고, 서비스의 잠재적인 가치를 즉각적으로 인식하게 해야한다.
*   모든 인터랙션 시점에 대해 시스템의 상태를 사용자에게 명확히 알려줘야 한다. 신뢰성과 사용성을 동시에 높이는 가장 좋은 방법이다.(현재 비주얼과 애니메이션이 정적인 느낌)
    * 로딩스피너
    * 검색이나 로딩 중 입력 input 비활성화
    * 목표 달성 시 축하(?) 애니메이션
*   메인 페이지의 텍스트 들을 감성적인 텍스트로 변경해야 할 필요가 있다. 서비스가 제공하는 이점을 복잡한 설명 대신 명확하게 제시해야 한다.
    * as is : 게임을 탐색하세요, 자연어로 질문하면 ai가 게임을 추천하고 분석합니다, ai기반 게임 추천-분석-리뷰-게심시장 정보
    * to be : ㅇㅇ님, 어떤 게임을 하고 계신가요?, 당신의 잠재력을 키워보세요! 등등

### 2. 사용자가 서비스의 기능과 워크플로우를 혼란 없이 이해하고, 자신이 원하는 목표 달성 방법을 제시한다.(온보딩)
*   step-by-step가이드, 온보딩을 통해 한 번에 모든 것을 보여주지 않고, 단계별로 진행하도록 유도한다. (무조건적인건 아님)
*   사용자가 목표를 달성하는 경로를 시각화 하는것을 집중해야한다. (이쁘게 보인는게 아닌 '사용자가 어떻게 성공할까'에 대한 답을 시각화)

### 3. 사용자가 학습 한 내용을 바탕으로 실제 행동을 취하도록 유도하고 지속적인 참여을 이끌어 내야한다.
*   학습된 내용을 즉시 적용 할 수 있는 명확한 액션을 제공해야 한다.(내 데이터에 어떻게 적용할지 등 다음단계 제안)
