## 1. 핵심 UX/UI 요소별 분석 및 개선 방안

UX/UI의 8가지 핵심 요소를 기준으로 현재 프로젝트의 구조적 잠재력을 분석하고 개선 방향 도출.

### 1. 유용성 (Useful) & 가치성 (Valuable) 확보
목표: 사용자가 앱을 사용하는 핵심 목표(예: 게임 성과 분석, 감성 데이터 이해)를 가장 쉽고 빠르게 달성하도록 지원해야 합니다.

*   분석: 프로젝트는 데이터 집계(game_metrics, sentiment_analyzer 등)와 분석 도구(radar_chart_runner, weighted_score_runner)를 포함하고 있습니다. 가치성을 높이려면, 단순히 데이터를 보여주는 것을 넘어 '행동 유도(Call to Action)'와 '통찰력(Insight)'을 제공해야 합니다.
*   개선 방향:
    *   핵심 가치 강조: 사용자가 보기에 가장 중요한 지표(KPI)를 대시보드 최상단에 배치하고, 이 지표가 사용자의 목표 달성에 어떻게 기여하는지 명확히 설명해야 합니다.
    *   맥락 제공: 단순한 수치 나열 대신, 수치 변화의 이유(Why)를 설명하는 텍스트(예: "지난주 대비 15% 상승, 긍정 키워드 증가")를 함께 제공하여 데이터의 유용성을 극대화합니다.

### 2. 사용성 (Usable) 확보
목표: 사용자가 복잡한 분석 기능이나 데이터 입력 과정에서 혼란을 느끼지 않고 목표를 달성하도록 직관적인 워크플로우를 구축해야 합니다.

*   분석: 파일 구조(pages/, components/, hooks/)는 컴포넌트 기반 설계가 잘 되어 있음을 시사합니다. 하지만 데이터 시각화(차트, 그래프)와 복잡한 설정(API 호출, 필터링)이 많을 경우, 탐색 가능성이 저해될 수 있습니다.
*   개선 방향:
    *   단순화된 워크플로우: 복잡한 분석 과정은 단계별(Step-by-Step) 프로세스로 나누어 사용자가 한 번에 너무 많은 정보를 처리하지 않도록 해야 합니다.
    *   명확한 피드백: 데이터 로딩 중이거나 분석 중일 때는 반드시 시각적인 로딩 인디케이터(스켈레톤 UI 등)와 명확한 텍스트 피드백을 제공하여 사용자의 대기 시간을 관리해야 합니다.

### 3. 탐색 가능성 (Findable) 확보
목표: 사용자가 원하는 정보나 기능에 최소한의 클릭으로 접근할 수 있도록 정보 구조를 체계화해야 합니다.

*   분석: 메뉴 구조(Navbar, Pagination)와 데이터 접근 방식이 중요합니다.
*   개선 방향:
    *   일관된 정보 구조: 모든 데이터 분석 기능과 설정 옵션은 일관된 위치에 배치되어야 합니다. 예를 들어, '게임 분석'과 '감성 분석'이 명확히 분리되어야 합니다.
    *   강력한 검색 기능: 데이터셋이 확장될 경우, 필터링 및 검색 기능(Search.jsx 등)을 고도화하여 사용자가 원하는 특정 데이터 포인트에 즉시 도달할 수 있도록 해야 합니다.

### 4. 신뢰성 (Credible) 확보
목표: 제공되는 모든 데이터와 분석 결과에 대한 사용자의 믿음을 확보해야 합니다.

*   분석: 데이터의 출처(Source)와 분석 방법론(Methodology)이 신뢰의 핵심입니다.
*   개선 방향:
    *   데이터 출처 명시: 표시되는 모든 지표 옆에 데이터가 어디서 왔는지(예: Steam 리뷰, 유튜브 시그널 등)를 명시하여 데이터의 신뢰성을 높여야 합니다.
    *   투명성: 분석 알고리즘이나 모델의 작동 방식에 대한 간략한 설명을 제공하여, 결과가 어떻게 도출되었는지에 대한 투명성을 확보합니다.

### 5. 사용성 테스트 & 지속적 개선 (UCD 접근법)
목표: 사용자의 실제 행동과 피드백을 통해 지속적으로 제품을 개선해야 합니다.

*   분석: 현재는 정적인 구조만 존재합니다. 개선은 반복적인 테스트를 통해 이루어져야 합니다.
*   개선 방향:
    *   A/B 테스트 준비: 특정 레이아웃, 버튼 위치, 데이터 표시 방식에 대해 A/B 테스트를 설계할 수 있도록 컴포넌트화(Componentization)를 철저히 해야 합니다.
    *   피드백 루프 구축: 사용자가 특정 분석 결과를 보고 "이 데이터가 부족하다"고 느낄 때, 이를 쉽게 피드백할 수 있는 기능을 추가하는 것을 고려해야 합니다.

### 6. UI 일관성 & 디자인 시스템 구축
목표: 모든 화면에서 동일한 경험을 제공하여 인지 부하를 줄이고 브랜드의 통일성을 확보합니다.

*   분석: 현재 GameCard.jsx, Navbar.jsx 등 컴포넌트가 존재하므로, 이들을 기반으로 디자인 시스템을 구축하는 것이 가장 시급합니다.
*   개선 방향:
    *   디자인 토큰 정의: 색상 팔레트, 타이포그래피 스케일, 간격(Spacing), 버튼 스타일(Primary, Secondary 등)에 대한 명확한 디자인 토큰을 정의하고 이를 Theme 또는 Style 파일에 통합해야 합니다.
    *   컴포넌트 표준화: GameCard와 같은 핵심 컴포넌트의 디자인을 표준화하여, 모든 페이지에서 동일한 시각적 언어를 사용하도록 강제해야 합니다.

### 7. 접근성 (Accessible) 확보
목표: 모든 사용자가 물리적 제약 없이 콘텐츠에 접근하고 상호작용할 수 있도록 설계해야 합니다.

*   분석: 이는 시각적 요소뿐만 아니라 코드 레벨의 고려가 필요합니다.
*   개선 방향:
    *   색상 대비(Contrast): index.css 및 모듈 CSS에서 텍스트와 배경 간의 대비 비율(WCAG 기준)을 반드시 확인해야 합니다.
    *   키보드 내비게이션: 모든 상호작용 요소(버튼, 링크, 입력 필드)가 키보드(Tab, Enter)로 접근 가능하도록 tabindex 및 포커스 스타일을 명확히 설정해야 합니다.

### 8. 감성 UX (Emotional Design) 확보
목표: 사용자와 시스템 간의 상호작용에서 긍정적인 감정(만족감, 몰입감)을 유발해야 합니다.

*   분석: 데이터 분석 앱은 특히 몰입감과 성취감을 제공할 때 감성적 연결이 강해집니다.
*   개선 방향:
    *   미묘한 애니메이션: 데이터가 로드되거나 차트가 업데이트될 때, 갑작스러운 변화 대신 부드럽고 의미 있는 애니메이션(예: 데이터가 점진적으로 채워지는 효과)을 사용하여 몰입도를 높여야 합니다.
    *   긍정적 피드백: 목표를 달성했을 때(예: 목표 점수 달성 시) 명확하고 기분 좋은 시각적/청각적 피드백을 제공하여 성취감을 강화해야 합니다.
